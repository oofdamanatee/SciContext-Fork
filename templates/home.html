SciContext: A JSON-LD Context File Development System

Semantic data is becoming an important part of the data publishing landscape. Any encoding of RDF (Resource Description Framework) requires that you annotate your data, so it can be converted into RDF triples - subject, predicate, object statements of knowledge. For the JSON for linked data encoding (JSON-LD) of RDF, semantic annotation of data is provided through the inclusion of context files. If you are developing JSON-LD files this can be an issue as not only do you need to create them, you need to make them available online so the conversion to RDF can be done. This Django project allows you to create, manage, and deploy any/all context files you need for your application(s).

This software is a web-based processing system, built in Django, to manage the creation and publishing of JSON-LD context files for JSON-LD data. As JSON-LD files are an encoding of the resource description framework (RDF), context files are needed to provide the semantic annotation (meaning) of data defined in a JSON-LD file, thus allowing them to be converted into RDF triples.

This application can be used for creation of context files for any JSON-LD data. Additionally, you can publish the context files to the GitHub pages branch of your GitHub repository, making them available to your JSON-LD.

How the system is set up

As there are many ontologies that have been developed, there are many places online that you might need to go to. Thankfully, there are now some centralized websites where ontologies are not only aggregated, but also indexed and searchable. SciContext takes advantage of these ontology 'lookup' servers as the source of the defined concepts in the ontologies they aggregate.

So, this application provides access (dropdown menu above) to the following resources to allow you to create and publish your JSON-LD context files:

  • Contexts: The context files that you create.
  • Fields: Fields are metadata entries that you define and add to your context files.
  • Ontologies: General information about ontologies and the domains they define.
  • Projects: Projects help organize your context files. Useful, but not required.
  • Servers: Ontology servers where you can search for definitions of concepts.
  • Terms: Specific concepts from ontologies that you want to use to create entries in your context files.
